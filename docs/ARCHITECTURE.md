## Configuration Flow

1. **User enables weather** (default: true in chat-ui.tsx)
2. **Sent to API** via `enableWeather: true` in request body
3. **Tool registered** if `enableWeather === true`
4. **Available to model** as part of tools object

## Performance Considerations

1. **API Calls**: Two sequential calls (geocode â†’ weather)
   - Average latency: 200-500ms total
   - Handled by tool execution, doesn't block UI

2. **Component Rendering**: WeatherCard is static
   - No animations or heavy computations
   - Pure presentation component

3. **Data Size**: Weather response ~2KB
   - Minimal impact on message size
   - Efficiently serialized in JSON
